1. Code handling storage based on URI:

fun insertImageToStore(fileInfo: FileInfo?) {
    fileInfo?.let {
        insertToStore(
            fileInfo,
            mediaContentHelper.getImageContentUri(),
            mediaContentHelper.generateImageContentValues(it)
        )
    }
}

private fun insertToStore(fileInfo: FileInfo, contentUri: Uri, contentValues: ContentValues) {
    executor.execute {
        val insertedUri = contentResolver.insert(contentUri, contentValues)
        insertedUri?.let { uri ->
            contentResolver.openInputStream(fileInfo.uri)?.use { input ->
                contentResolver.openOutputStream(uri)?.use { output ->
                    input.copyTo(output)
                }
            }
        }
    }
}

2. FileInfo.kt attributes:
	1. (uri): Refers to the temporary URI generated by FileProvider where the camera saves the captured image.

	2. (relativePath): Refers to the target path in MediaStore relative to standard directories (e.g., `Environment.DIRECTORY_PICTURES`) where the file will be stored.

	3. Chronological order from capture to MediaStore:
		1. User clicks the "Take Photo" button in `MainActivity`.
		2. openImageCapture() calls providerFileManager.generatePhotoUri(time), generating a FileInfo with a temporary URI and file path.

	4. The temporary URI is launched via takePictureLauncher.launch(photoInfo.uri) and the camera saves the image there.
	
	5. After capture, the ActivityResultLauncher callback is triggered.

	6. ProviderFileManager.insertImageToStore(photoInfo) is called.
		- insertToStore inserts the file into MediaStore and copies content from the temporary URI.

	7. The image is now stored in MediaStore and accessible in gallery apps.
